# REST API для push-уведомлений

Апи имеет следующие маршруты:

- POST `/auth/login`
- GET `/notifications`

## Аутентификация

Аутентификация в системе происходит по средствам ввода
логина и пароля. Необходимо отправить **HTTP POST** запрос
на конечную точку `/auth/login`, передав тело следующего содержания:

```json
{
  "username": "имя пользователя",
  "password": "пароль пользователя"
}
```

Если данные подойдут, сервер вернет следующий ответ:

```json
{
  "token": "подписанный секретным ключем сервера JWT-токен, содержащий id пользователя"
}
```

Если данные не верные (пользователя не существует, либо пароль не подошел),
сервер вернет следующий ответ с HTTP кодом 401:

```json
{
  "error": "Неверные данные",
  "code": 400
}
```

## Запрос новых уведомлений

Для запроса новых уведомлений необходимо отправить
HTTP GET запрос на конечную точку `/notifications`,
передав при этом заголовок следующего вида:

```yaml
Authorization: Bearer <ваш токен здесь>
```

Сервер прорит токен на валидность и извлечет из него
ID пользователя. В случае успеха, пользователь получит
ответ следующего вида:

```json
[
  {
    "date": "2026-06-25T03:43:40.271Z",
    "title": "Сообщение от Васи",
    "icon": "https://example.com/icon.png",
    "priority": "high",
    "text": "Жду тебя на гаражах"
  },
  {
    "date": "2026-06-25T03:53:40.271Z",
    "title": "Сообщение от Васи",
    "icon": "https://example.com/icon.png",
    "priority": "high",
    "text": "Ну и где ты?"
  }
]
```

В обратном случае, пользователь получит ответ слудеющего вида:

```json
{
  "error": "Вы не авторизованы",
  "code": 401
}
```

Приложение: схема API в формате OpenAPI
